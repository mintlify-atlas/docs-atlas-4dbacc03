## Key files

### `app.js`

The application entry point. It:

- Creates the Express app instance
- Registers `body-parser` middleware for URL-encoded form data
- Configures `express-session` for login state
- Sets EJS as the view engine and points Express at the `views/` and `public/` directories
- Imports all controllers and mounts every route
- Calls `sequelize.sync()` before starting the HTTP server

### `config/Conexion.js`

Creates a single shared Sequelize instance connected to the `sistema_viajes` MySQL database. Every model file imports this instance directly with `require('../config/Conexion')`. See the [Configuration](/developer/configuration) page for full details.

### `models/relaciones.js`

Defines all Sequelize associations in one place via `applyAssociations(sequelize)`. This keeps relationship logic out of individual model files. The associations are:

- `rol` ‚Üí `usuarios` (one-to-many via `rl_id`)
- `usuarios` ‚Üí `viajes` (one-to-many via `us_id`)
- `sucursales` ‚Üí `viajes` (one-to-many via `sc_id`)
- `colaboradores` ‚Üí `viajes` (one-to-many via `cl_id`)
- `transportistas` ‚Üí `viajes` (one-to-many via `tr_id`)
- `sucursales` ‚Üî `colaboradores` (many-to-many through `colaborador_sucursal`)

### `models/index.js`

An alternative model bootstrapper that reads the database URL from `process.env.DB_URL` and registers all models and associations. This file is not currently imported by `app.js` ‚Äî the production app connects via `config/Conexion.js` directly.

## MVC pattern

TRC follows a Model‚ÄìView‚ÄìController pattern:

| Layer | Directory | Responsibility |
|---|---|---|
| **Model** | `models/` | Sequelize model definitions and database associations. Provides the data access interface used by controllers. |
| **View** | `views/` | EJS templates that render HTML. They receive data as variables passed by controllers via `res.render()`. |
| **Controller** | `controllers/` | Request handlers. Read incoming data from `req.body` / `req.params`, call models, and either render a view or return JSON. |
| **Router** | `app.js` | Route declarations that map HTTP methods and paths to controller functions. |

## Controllers

| File | Entity | Operations |
|---|---|---|
| `loginController.js` | Session / users | Authenticates a user, stores the user record in the session, and redirects based on role. |
| `colaboradorController.js` | Employees | List active employees, create with branch assignment (transactional), update, soft-delete (sets `cl_estado = 0`). |
| `sucursalController.js` | Branches | List active branches, create, update, soft-delete (sets `sc_estado = 0`). |
| `viajesController.js` | Trips | Fetch collaborators by branch (JSON), fetch distance for a collaborator/branch pair (JSON), create a trip record. |
| `transportistaController.js` | Transporters | List active transporters, create, update, soft-delete (sets `tr_estado = 0`). |
| `reportesController.js` | Reports | Run a JOIN query across `viajes`, `usuarios`, `sucursales`, and `transportistas` and render the results. |

<Note>
  All delete operations in TRC are **soft deletes** ‚Äî they set a `_estado` flag to `0` rather than removing the row. Active records have `_estado = 1`.
</Note>
