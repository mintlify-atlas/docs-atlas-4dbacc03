**Branches (sucursales)** are the physical company locations that employees travel to. Each branch has a name and address, and collaborators are linked to branches via the `colaborador_sucursal` junction table with a recorded distance. When a trip is created, the user selects a destination branch to determine which collaborators are available for selection.

## Data Model

| Field | Type | Constraints | Description |
|---|---|---|---|
| `sc_id` | `INT` | PK, auto-increment, NOT NULL | Unique branch identifier |
| `sc_nombre` | `VARCHAR(50)` | NOT NULL | Branch name |
| `sc_direccion` | `TEXT` | NOT NULL | Branch address |
| `sc_estado` | `TINYINT(1)` | NOT NULL | `1` = active, `0` = soft-deleted |

```sql
CREATE TABLE `sucursales` (
  `sc_id`        int(11)     NOT NULL AUTO_INCREMENT,
  `sc_nombre`    varchar(50) NOT NULL,
  `sc_direccion` text        NOT NULL,
  `sc_estado`    tinyint(1)  NOT NULL,
  PRIMARY KEY (`sc_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## Soft-Delete Pattern

Deleting a branch sets `sc_estado = 0`. The branch record is retained in the database so that existing trip records and collaboratorâ€“branch assignments remain valid.

```javascript
// Delete handler â€” sets sc_estado to 0 instead of running DELETE
await sucursal.update(
  { sc_estado: 0 },
  { where: { sc_id: id } }
);
```

| `sc_estado` | Meaning |
|---|---|
| `1` | Active â€” visible in branch lists and the trip creation form |
| `0` | Inactive â€” hidden from UI, historical records preserved |

## Trip Form Filtering

The `GET /viajes` route explicitly filters for active branches when loading the trip creation view:

```javascript
// app.js â€” only active branches are passed to the trip form
const sucursales = await sucursal1.findAll({
  where: {
    sc_estado: 1,
  }
});
```

<Note>
  The `/viajes` route is protected by a session check (`rl_id == 1`). Only users with the Store Manager role can access the trip creation form and see the branch list.
</Note>

## API Routes

| Method | Route | Description |
|---|---|---|
| `GET` | `/sucursales` | Render the branches list (only `sc_estado = 1` records) |
| `POST` | `/sucursales/create` | Create a new branch |
| `GET` | `/sucursales/update/:id` | Return a single branch record as JSON for editing |
| `POST` | `/sucursales/update` | Update a branch's name and address |
| `POST` | `/sucursales/delete/:id` | Soft-delete a branch by setting `sc_estado = 0` |
