**Transporters (transportistas)** are the drivers or carriers contracted to handle employee trips. Each transporter has a name and a flat fare rate (`tr_tarifa`) that represents their standard charge per trip. When creating a trip, the user selects an active transporter and enters the actual cost â€” which may reference the transporter's tariff.

## Data Model

| Field | Type | Constraints | Description |
|---|---|---|---|
| `tr_id` | `INT` | PK, auto-increment | Unique transporter identifier |
| `tr_nombre` | `VARCHAR(50)` | NOT NULL | Full name of the transporter |
| `tr_tarifa` | `DECIMAL(7,2)` | NOT NULL | Standard fare rate per trip in Lempiras |
| `tr_estado` | `TINYINT(1)` | NOT NULL | `1` = active, `0` = soft-deleted |

```sql
CREATE TABLE `transportistas` (
  `tr_id`     int(11)      NOT NULL AUTO_INCREMENT,
  `tr_nombre` varchar(50)  NOT NULL,
  `tr_tarifa` decimal(7,2) NOT NULL,
  `tr_estado` tinyint(1)   NOT NULL,
  PRIMARY KEY (`tr_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## Soft-Delete Pattern

Deleting a transporter sets `tr_estado = 0`. The record remains in the database so that all existing trip records referencing `tr_id` continue to resolve correctly.

```javascript
// Delete handler â€” sets tr_estado to 0 instead of running DELETE
await transportista.update(
  { tr_estado: 0 },
  { where: { tr_id: id } }
);
```

| `tr_estado` | Meaning |
|---|---|
| `1` | Active â€” visible in transporter lists and the trip creation form |
| `0` | Inactive â€” hidden from UI, historical trip records preserved |

## Trip Form Filtering

The `GET /viajes` route filters for active transporters when rendering the trip creation page:

```javascript
// app.js â€” only active transporters are passed to the trip form
const transportista = await transportistas.findAll({
  where: {
    tr_estado: 1,
  }
});
```

<Note>
  The `tr_tarifa` value is available in the trip form as a reference. The actual trip cost (`vj_costo`) is entered manually by the user and may differ from the standard tariff if special arrangements apply.
</Note>

## API Routes

| Method | Route | Description |
|---|---|---|
| `GET` | `/transportistas` | Render the transporters list (only `tr_estado = 1` records) |
| `POST` | `/transportistas/create` | Create a new transporter |
| `GET` | `/transportistas/update/:id` | Return a single transporter record as JSON for editing |
| `POST` | `/transportistas/update` | Update a transporter's name and fare rate |
| `POST` | `/transportistas/delete/:id` | Soft-delete a transporter by setting `tr_estado = 0` |
