The **Reports (Reportes)** view provides managers with a unified, read-only overview of every trip recorded in the system. It joins data from all four core entities — users, branches, transporters, and trips — into a single flat result set, making it straightforward to audit travel activity and monitor costs.

## Route

| Method | Route | Description |
|---|---|---|
| `GET` | `/reportes` | Render the full trips report view |

## Report Columns

The following columns are displayed in the report view:

| Column | Source Field | Description |
|---|---|---|
| User | `us.us_nombre` | Name of the user who registered the trip |
| Branch | `suc.sc_nombre` | Name of the destination branch |
| Transporter | `tr.tr_nombre` | Name of the transporter who handled the trip |
| Cost | `vj.vj_costo` | Total trip cost in Lempiras |
| Distance | `vj.vj_recorrido` | Distance traveled in kilometers |
| Date | `DATE(vj.vj_fecha)` | Date the trip was recorded (time component stripped) |

## SQL Query

The report is generated by a single JOIN query across all four tables:

```sql
SELECT
  us.us_nombre,
  suc.sc_nombre,
  tr.tr_nombre,
  vj.vj_costo,
  vj.vj_recorrido,
  DATE(vj.vj_fecha) AS vj_fecha
FROM viajes AS vj
INNER JOIN usuarios AS us
  ON us.us_id = vj.us_id
INNER JOIN sucursales AS suc
  ON suc.sc_id = vj.sc_id
INNER JOIN transportistas AS tr
  ON tr.tr_id = vj.tr_id;
```

<Note>
  The query uses `DATE(vj.vj_fecha)` to strip the time component from the `TIMESTAMP` column, so only the calendar date is displayed in the report.
</Note>

## How to Use the Report

<Steps>
  <Step title="Navigate to /reportes">
    Open the reports page. All trip records are loaded immediately — no filters are applied by default.
  </Step>
  <Step title="Review Trip Costs">
    Scan the `vj_costo` column to identify the most expensive trips. Compare costs against each transporter's standard tariff (`tr_tarifa`) to spot discrepancies.
  </Step>
  <Step title="Monitor Distance">
    Check `vj_recorrido` to verify that distances are within the system's allowed range (1–150 km). Unusually high values may warrant review.
  </Step>
  <Step title="Track by Date">
    Use the `vj_fecha` column to group trips by day or period, giving a time-based view of travel frequency and total expenditure.
  </Step>
  <Step title="Identify High-Traffic Branches">
    Review `suc.sc_nombre` to see which branches have the most trips, helping with resource allocation and transporter scheduling.
  </Step>
</Steps>

## Implementation Notes

<Info>
  The report query returns **all trips regardless of status**. Trip records with any `vj_estado` value are included. If filtering by active trips only is needed, add `WHERE vj.vj_estado = 1` to the query.
</Info>

<Tip>
  To export report data, consider adding a CSV download endpoint that runs the same query and streams the result as a file. The `vj_fecha` column already normalizes dates, making it straightforward to import into spreadsheet tools.
</Tip>
