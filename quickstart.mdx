This guide walks you through getting TRC running on your machine from scratch. By the end you will be logged in as the sample admin user and able to create a viaje.

## Prerequisites

Make sure you have the following installed before you start:

- **Node.js** v18 or later ŌĆö [nodejs.org](https://nodejs.org)
- **MySQL** 8.0 or later ŌĆö the application requires a running MySQL server with a user that has permission to create and modify databases

<Tip>
  On macOS, MySQL can be installed with `brew install mysql`. On Ubuntu/Debian, use `sudo apt install mysql-server`.
</Tip>

## Steps

<Steps>
  <Step title="Clone the repository">
    Clone the TRC repository from GitHub and enter the project directory.

    ```bash
    git clone https://github.com/molina071/TRC.git
    cd TRC
    ```
  </Step>

  <Step title="Install dependencies">
    Install all Node.js packages listed in `package.json`.

    ```bash
    npm install
    ```

    This installs Express 5, Sequelize, EJS, express-session, bcrypt, and the MySQL2 driver.
  </Step>

  <Step title="Create the MySQL database">
    Create the `sistema_viajes` database and import the provided SQL schema and seed data.

    ```bash
    mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS sistema_viajes;"
    mysql -u root -p sistema_viajes < BD/sistema_viajes.sql
    ```

    The SQL file creates all tables (`usuarios`, `viajes`, `colaboradores`, `sucursales`, `transportistas`, `roles`) and loads the sample records including the two test users.

    <Note>
      If your MySQL root user has no password, omit the `-p` flag: `mysql -u root`.
    </Note>
  </Step>

  <Step title="Configure the database connection">
    Open `config/Conexion.js` and update the Sequelize connection to match your MySQL credentials.

    ```javascript config/Conexion.js
    const { Sequelize } = require('sequelize');

    const sequelize = new Sequelize('sistema_viajes', 'root', '', {
        host: "localhost",
        dialect: 'mysql',
    });

    module.exports = sequelize;
    ```

    The three positional arguments are:

    1. **Database name** ŌĆö `sistema_viajes` (matches the database you created in the previous step)
    2. **Username** ŌĆö `root` by default; change to your MySQL username
    3. **Password** ŌĆö empty string by default; replace with your MySQL password

    <Warning>
      Do not commit real credentials to version control. Consider using environment variables for production deployments.
    </Warning>
  </Step>

  <Step title="Start the server">
    Start the Express application. Sequelize will synchronize the models before the server begins listening.

    ```bash
    node app.js
    ```

    You should see:

    ```
    Server is running on port 3000
    ```

    The application listens on port 3000 by default. You can override this with the `PORT` environment variable:

    ```bash
    PORT=8080 node app.js
    ```
  </Step>

  <Step title="Log in as the sample admin user">
    Open [http://localhost:3000](http://localhost:3000) in your browser. You will see the TRC login page.

    Enter the following credentials for the sample **Gerente de Tienda** account:

    | Field | Value |
    |---|---|
    | Correo | `jpenia55@gmail.com` |
    | Contrase├▒a | `nancy` |

    After a successful login you will be redirected to `/dashboardAdmin`, the admin dashboard.

    <Tip>
      A second test account is available for the **Usuario** role: correo `7josueortega111@gmail.com`, password `josue`. This account has limited access and will be redirected to `/dashboard`.
    </Tip>
  </Step>

  <Step title="Navigate to the admin dashboard">
    The admin dashboard (`/dashboardAdmin`) is the starting point for all management tasks. From here you can access:

    - **Viajes** ŌĆö register and review trips
    - **Colaboradores** ŌĆö manage the collaborator directory
    - **Sucursales** ŌĆö manage branch locations
    - **Transportistas** ŌĆö manage transport providers
    - **Reportes** ŌĆö view consolidated trip reports
  </Step>

  <Step title="Create your first viaje">
    Navigate to [http://localhost:3000/viajes](http://localhost:3000/viajes).

    <Note>
      This route is restricted to users with `rl_id = 1` (Gerente de Tienda). Accessing it while logged in as a Usuario returns a 403 error.
    </Note>

    Fill in the trip form:

    - **Colaborador** ŌĆö select the employee making the trip (identified by cedula)
    - **Sucursal** ŌĆö select the destination branch (only active branches are shown)
    - **Transportista** ŌĆö select the transport provider (only active transporters are shown)
    - **Recorrido** ŌĆö distance in kilometers

    The trip cost (`vj_costo`) is calculated automatically from the transporter's fare (`tr_tarifa`) and the distance. Submit the form to save the viaje.

    <Check>
      Once submitted, the new viaje will appear in the reports at `/reportes`.
    </Check>
  </Step>
</Steps>
